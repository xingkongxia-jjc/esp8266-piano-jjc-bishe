# 按键按下发声详解

这篇文档讲的是：为什么按下一个键以后，蜂鸣器就会发出对应的音。

这个模块的核心思想其实不复杂：

1. 每个琴键对应一个音符编号。
2. 每个音符编号对应一个频率。
3. 程序把这个频率交给蜂鸣器。
4. 蜂鸣器按这个频率振动，于是你听到声音。

## 1. 相关文件

发声功能主要集中在：

- `src/audio.cpp`
- `src/audio.h`
- `src/config.h`
- `src/main.cpp`

它们分别负责：

- `audio.cpp`：音高换算、单音播放、多键播放、停止播放
- `audio.h`：对外声明变量和函数
- `config.h`：默认音长、八度偏移、混音更新间隔
- `main.cpp`：在检测到按键后调用发声函数

## 2. 硬件上是怎么发声的

项目里的发声引脚是：

```cpp
#define BUZZER_PIN 12
```

也就是 ESP8266 的 GPIO12 接了蜂鸣器。

初始化函数非常简单：

```cpp
void initAudio() {
  pinMode(BUZZER_PIN, OUTPUT);
}
```

这说明音频模块本质上是把 GPIO12 当成输出口来驱动。

## 3. 为什么频率会对应音高

声音本质上是振动。

如果一个器件每秒振动 262 次，人耳会听到大约是 `C4`。如果每秒振动 440 次，人耳会听到 `A4`。

所以程序只要控制“每秒振动多少次”，就能决定音高。

项目里用一个数组保存基础频率：

```cpp
int baseFrequencies[] = {
  0,
  262, 277, 294, 311, 330, 349,
  370, 392, 415, 440, 466, 494
};
```

它表示一个八度内 12 个半音：

- 1 -> C
- 2 -> C#
- 3 -> D
- 4 -> D#
- 5 -> E
- 6 -> F
- 7 -> F#
- 8 -> G
- 9 -> G#
- 10 -> A
- 11 -> A#
- 12 -> B

## 4. 主流程里什么时候开始发声

在 `main.cpp` 中，按键检测和发声连接在一起：

```cpp
if (keyCount > 0) {
  playMultipleNotes(pressedKeys, keyCount);
} else {
  stopAllAudio();
}
```

意思很直接：

- 如果当前有按键，就播放
- 如果当前没有按键，就停掉所有声音

也就是说，音频模块不是自己主动工作的，而是由按键检测结果驱动的。

## 5. 单音是怎么播放的

真正的播放入口是 `playMultipleNotes()`：

```cpp
void playMultipleNotes(int* keys, int keyCount) {
  isMixPlaying = false;
  noTone(BUZZER_PIN);

  if (keyCount == 0) {
    return;
  }

  if (keyCount == 1) {
    tone(BUZZER_PIN, getNoteFrequency(keys[0]), noteDuration);
  } else {
    ...
  }
}
```

当只按下一个键时，会直接调用：

```cpp
tone(BUZZER_PIN, getNoteFrequency(keys[0]), noteDuration);
```

这行可以理解成：

- 在 `BUZZER_PIN`
- 输出某个频率
- 持续 `noteDuration` 毫秒

这是最直接的发声方式。

## 6. `getNoteFrequency()` 做了什么

这个函数的职责是：

“把按键编号变成真实播放频率”。

代码如下：

```cpp
int getNoteFrequency(int noteIndex) {
  if (noteIndex < 1 || noteIndex > 12) return 0;

  float freq = baseFrequencies[noteIndex];
  for (int i = 0; i < abs(octaveShift); i++) {
    if (octaveShift > 0) {
      freq *= 2.0;
    } else if (octaveShift < 0) {
      freq /= 2.0;
    }
  }
  return (int)freq;
}
```

## 7. 为什么乘 2 或除 2 就是换八度

音乐里有一个很重要的规律：

- 升高一个八度，频率变成原来的 2 倍
- 降低一个八度，频率变成原来的 1/2

比如：

- `A4 = 440Hz`
- `A5 = 880Hz`
- `A3 = 220Hz`

所以这里的 `octaveShift` 就是在做这个变换。

例如：

- `octaveShift = 0`，按原频率播放
- `octaveShift = 1`，频率翻倍
- `octaveShift = -1`，频率减半

## 8. 多键同时按下时为什么还能发声

理论上，真正的和弦播放需要同时输出多个频率。但一个普通蜂鸣器加 `tone()` 函数，不能像真正音源那样高质量地同时合成很多音。

这个项目用了一个“简化混音”的方法。

### 8.1 多键播放入口

当 `keyCount > 1` 时：

```cpp
mixKeyCount = keyCount;
mixStartTime = millis();
lastMixUpdate = micros();

for (int i = 0; i < keyCount; i++) {
  mixKeys[i] = keys[i];
}

isMixPlaying = true;
updateMixedTone();
```

这段逻辑不是立刻生成真正的多路波形，而是：

1. 记住当前有哪些键被按下
2. 开启混音状态 `isMixPlaying`
3. 后续在主循环里不断调用 `updateMixedTone()` 维护播放

### 8.2 主循环如何维护混音

在 `main.cpp` 里：

```cpp
if (isMixPlaying) {
  unsigned long currentMicros = micros();

  if (currentMicros - lastMixUpdate >= MIX_UPDATE_INTERVAL) {
    updateMixedTone();
    lastMixUpdate = currentMicros;
  }
}
```

这表示只要处于混音状态，程序就会周期性刷新声音。

### 8.3 混音是怎么“简化”的

`updateMixedTone()` 的核心是：

```cpp
float avgFrequency = 0;
for (int i = 0; i < mixKeyCount; i++) {
  avgFrequency += getNoteFrequency(mixKeys[i]);
}
avgFrequency /= mixKeyCount;

tone(BUZZER_PIN, (int)avgFrequency, 100);
```

也就是说，这里并没有真正把多个波形做叠加，而是：

- 先计算多个按键频率的平均值
- 再播放这个平均频率

## 9. 这种“平均频率混音”意味着什么

这是一种工程上的折中方案。

优点：

- 实现简单
- 资源占用低
- 对 ESP8266 + 蜂鸣器 这种硬件很友好

缺点：

- 听起来不是真正的和弦
- 多键时更像一个折中的音高
- 音色表现比较有限

所以你可以把它理解为：

“项目支持多键输入，也能给出声音反馈，但不是专业级的多声部合成器。”

对于学习型项目来说，这完全合理。

## 10. 声音什么时候停止

有两个主要停止方式。

### 10.1 没有按键时停止

```cpp
void stopAllAudio() {
  isMixPlaying = false;
  noTone(BUZZER_PIN);
  digitalWrite(BUZZER_PIN, LOW);
}
```

这里做了三件事：

1. 关闭混音状态
2. 停止 `tone()` 输出
3. 把引脚拉低，确保蜂鸣器不再响

### 10.2 到了音长自动停止

混音模式里还有时长判断：

```cpp
if (currentTime - mixStartTime > noteDuration) {
  isMixPlaying = false;
  digitalWrite(BUZZER_PIN, LOW);
  return;
}
```

这表示声音不会无限持续，而是受 `noteDuration` 控制。

## 11. 关键参数分别有什么用

### 11.1 `noteDuration`

默认定义在 `config.h`：

```cpp
#define DEFAULT_NOTE_DURATION 500
```

表示默认音长是 500ms。

影响：

- 数值越大，音更长
- 数值越小，声音更短促

### 11.2 `octaveShift`

默认值：

```cpp
#define DEFAULT_OCTAVE_SHIFT 0
```

表示默认不升不降八度。

### 11.3 `MIX_UPDATE_INTERVAL`

默认值：

```cpp
#define MIX_UPDATE_INTERVAL 100
```

配合 `micros()` 使用，单位是微秒。

它控制混音状态下，多久更新一次输出。

## 12. 这个模块里值得掌握的知识点

### 12.1 频率和音高的关系

频率越高，声音越尖。

频率越低，声音越沉。

这就是程序为什么只要控制频率，就能“弹”出不同音符。

### 12.2 `tone()` 函数的意义

`tone()` 并不是播放录音文件，它更像是在某个引脚上输出固定频率的方波。

所以它很适合：

- 蜂鸣器
- 简单提示音
- 基础电子乐音

但不适合高质量音色合成。

### 12.3 为什么蜂鸣器方案简单但常见

因为它成本低、接线简单、代码也简单，非常适合作为嵌入式入门项目。

如果以后你想升级效果，可以考虑：

- DAC 输出
- I2S 音频模块
- 真正的波表合成或 PCM 播放

## 13. 这个项目的发声链路，按步骤怎么理解

你可以把它记成下面 6 步：

1. 按键模块检测到键 1~12 中的某个键被按下。
2. 主程序把这个键号传给 `playMultipleNotes()`。
3. `playMultipleNotes()` 判断是单键还是多键。
4. `getNoteFrequency()` 把键号换算成频率。
5. `tone()` 在蜂鸣器引脚输出这个频率。
6. 蜂鸣器振动，人耳听到音高。

## 14. 如果你想继续改进音频模块

比较适合继续练手的方向有：

1. 把平均频率混音改成快速轮询多个频率的伪复音。
2. 给每个音符加入包络，让声音不那么生硬。
3. 改成真正的音频输出模块，而不是简单蜂鸣器。
4. 为不同模式设置不同音长。

## 15. 一句话总结

这个项目的发声逻辑，本质上是：

“把按键编号映射为频率，再用 `tone()` 驱动蜂鸣器输出这个频率；单键直接播放，多键则用平均频率做简化混音。”

理解这句话，你就抓住这个模块的核心了。
