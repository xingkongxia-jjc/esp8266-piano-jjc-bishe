# EDA-Piano 文档说明

这个目录用于补充项目中几个核心功能的实现说明，内容尽量按“小白也能顺着看懂”的方式来写。

如果你第一次接触这个项目，建议按下面顺序阅读：

1. `01-按键检测.md`
2. `02-按键按下发声.md`
3. `03-OLED显示.md`

## 这几个功能在项目里的关系

这个电子钢琴的核心链路可以先记成一句话：

“检测到按键 -> 解析出按了哪些音 -> 蜂鸣器发声 -> OLED 更新界面”。

对应到代码里，大致流程如下：

```text
setup()
  ├─ initAudio()    初始化蜂鸣器输出
  ├─ initTouch()    初始化触摸芯片和中断
  ├─ initDisplay()  初始化 OLED
  └─ 进入 loop()

loop()
  ├─ detectMultipleKeys()   读取触摸键值
  ├─ parseKeys()            把键值解析成 1~12 的音符编号
  ├─ playMultipleNotes()    根据按键结果发声
  └─ displayMultipleKeys()  在 OLED 上显示当前按键状态
```

## 建议先认识的几个文件

- `src/main.cpp`：主流程入口，最适合先看
- `src/touch.cpp`：按键检测
- `src/audio.cpp`：按键发声
- `src/display.cpp`：OLED 显示
- `src/config.h`：引脚、时间参数、默认配置
- `src/SC12B.cpp` / `src/SC12B.h`：触摸芯片底层驱动

## 先建立一个整体印象

这个项目不是“按一个普通机械按钮”，而是“通过电容触摸芯片检测手指接触电极”。

所以它的工作方式是：

1. SC12B 芯片负责检测 12 个触摸通道有没有被按下。
2. ESP8266 通过 I2C 从 SC12B 读取结果。
3. 程序把读取到的位图数据转换成“第几个琴键被按下”。
4. 再把这些琴键映射到不同频率，让蜂鸣器发出声音。
5. 最后把当前状态画到 128x32 的 OLED 屏幕上。

## 适合初学者的阅读方法

如果你看代码容易晕，可以照这个办法：

1. 先看 `main.cpp`，只看函数名，不纠结每一行。
2. 看到 `detectMultipleKeys()` 就去看按键文档。
3. 看到 `playMultipleNotes()` 就去看声音文档。
4. 看到 `displayMultipleKeys()` 就去看显示文档。
5. 最后再回来看 `config.h` 里的参数，理解系统是怎么调出来的。

## 你会在文档里反复看到的几个概念

- `I2C`：一种常见的芯片通信总线，OLED 和触摸芯片都在用。
- `中断`：外设一有变化，就通知 MCU 尽快处理。
- `防抖`：避免一次触摸被程序误判成多次。
- `位运算`：用二进制位表示多个按键状态。
- `tone()`：Arduino/ESP8266 常见的发声函数，用频率驱动蜂鸣器。
- `frame buffer`：先在内存里画好画面，再统一刷新到 OLED。

## 一句话总结

这套系统本质上就是一个“输入 -> 处理 -> 输出”的嵌入式小项目：

- 输入：触摸按键
- 处理：解析键值、判断状态、选择音高
- 输出：蜂鸣器发声 + OLED 显示

理解了这条链路，再看具体实现就会轻松很多。
