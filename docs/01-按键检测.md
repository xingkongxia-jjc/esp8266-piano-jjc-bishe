# 按键检测详解

这篇文档讲的是项目里“怎么知道你按了哪个琴键”。

如果用最简单的话来说，这个功能做了三件事：

1. 初始化触摸芯片和中断。
2. 读取触摸芯片返回的键值。
3. 把键值解析成程序能理解的按键编号。

## 1. 相关文件

按键检测主要在下面几个文件里：

- `src/touch.cpp`
- `src/touch.h`
- `src/SC12B.cpp`
- `src/SC12B.h`
- `src/config.h`
- `src/main.cpp`

它们的职责可以这样理解：

- `touch.cpp`：按键检测主逻辑
- `SC12B.cpp`：和触摸芯片通信的底层驱动
- `config.h`：放检测相关参数，比如防抖时间和采样间隔
- `main.cpp`：在主循环中调用按键检测

## 2. 硬件层面到底发生了什么

项目里不是普通按键开关，而是 `SC12B` 电容触摸芯片。

它的工作思路是：

1. 每个触摸按键都对应芯片的一个通道。
2. 手指接近或接触触摸电极时，电容值会发生变化。
3. SC12B 检测到这个变化后，把按键状态保存到寄存器里。
4. ESP8266 通过 I2C 去读取这些寄存器。
5. 程序再把读出来的二进制数据翻译成“按下了第几个键”。

你可以把 SC12B 理解成一个“专门负责检测触摸”的小助手，ESP8266 不需要自己直接测电容，只需要向它读结果。

## 3. 初始化是怎么做的

按键模块的初始化函数是 `initTouch()`：

```cpp
void initTouch() {
  pinMode(TOUCH_INTERRUPT_PIN, INPUT_PULLUP);
  attachInterrupt(digitalPinToInterrupt(TOUCH_INTERRUPT_PIN), checkkey, CHANGE);
  touchPannel.begin();
  touchPannel.init();
  applyTouchSensitivity();
}
```

这几行分别在做什么：

### 3.1 设置中断引脚

```cpp
pinMode(TOUCH_INTERRUPT_PIN, INPUT_PULLUP);
```

这里的 `TOUCH_INTERRUPT_PIN` 在 `config.h` 里定义为 `14`。

这说明：

- 触摸芯片有一根中断输出线接到了 ESP8266 的 GPIO14。
- `INPUT_PULLUP` 表示把这个引脚设置成输入，并启用内部上拉。

上拉的作用是让引脚在默认情况下保持稳定高电平，避免悬空乱跳。

### 3.2 注册中断函数

```cpp
attachInterrupt(digitalPinToInterrupt(TOUCH_INTERRUPT_PIN), checkkey, CHANGE);
```

这行代码的意思是：

- 只要这个中断引脚电平发生变化，
- 就立刻执行 `checkkey()` 这个中断服务函数。

对应的中断函数非常短：

```cpp
void IRAM_ATTR checkkey() {
  iftouch = true;
}
```

这里为什么只做一件小事，而不是直接读取按键？

因为中断函数里要尽量短、尽量快。中断里做太多复杂逻辑，容易引发系统不稳定。这个项目采取的是很常见的做法：

- 中断里只立一个标志位 `iftouch = true`
- 真正复杂的读取和处理，放回 `loop()` 里做

这是一种标准的“中断通知 + 主循环处理”设计。

### 3.3 初始化触摸芯片

```cpp
touchPannel.begin();
touchPannel.init();
```

`begin()` 最终会调用 `Wire.begin()`，也就是初始化 I2C 总线。

`init()` 会给 SC12B 写入默认配置，例如：

- 灵敏度寄存器
- 控制寄存器

也就是说，这一步是把触摸芯片带到一个可正常工作的状态。

### 3.4 应用触摸灵敏度

```cpp
applyTouchSensitivity();
```

这个函数根据变量 `touchSensitivity` 的值，选择 `LEVEL0` 到 `LEVEL15` 之间的某个等级，再写到 SC12B 的寄存器里。

灵敏度越高，不一定越好。太高可能更容易误触，太低又可能按了没反应。所以这个参数本质上是在“灵敏”和“稳定”之间取平衡。

## 4. 主循环里怎么检测按键

按键检测在 `main.cpp` 里有两条路径：

1. 周期性轮询
2. 中断触发后再读取

这也是这个项目比较稳的一点，因为它不是只靠一种机制。

### 4.1 轮询检测

主循环里有这样一段：

```cpp
if (currentTime - lastSampleTime > SAMPLE_INTERVAL) {
  uint16_t keyValue = detectMultipleKeys();

  if (keyValue != previousKeys && currentTime - lastKeyTime > KEY_DEBOUNCE_TIME) {
    ...
  }

  lastSampleTime = currentTime;
}
```

这里有两个重要参数：

- `SAMPLE_INTERVAL = 5ms`
- `KEY_DEBOUNCE_TIME = 20ms`

含义分别是：

- 每隔 5ms 才采样一次，不会无限高速乱读
- 两次有效变化之间至少间隔 20ms，用来防抖

### 4.2 中断驱动检测

主循环后面还有一段：

```cpp
if (iftouch) {
  iftouch = false;
  ...
}
```

意思是：

- 如果中断告诉程序“可能有触摸变化了”
- 那就尽快去再读一次按键

这样做的优点是：

- 响应更及时
- 不容易错过短暂触摸
- 比纯轮询更灵敏

## 5. 多键检测算法是怎么做的

项目中最核心的检测函数是 `detectMultipleKeys()`：

```cpp
uint16_t detectMultipleKeys() {
  uint16_t combinedKeys = 0;

  for (int i = 0; i < 5; i++) {
    uint16_t keyValue = touchPannel.getKeyValue();
    combinedKeys |= keyValue;
    delay(1);
  }

  return combinedKeys;
}
```

初学者最容易看不懂的是这句：

```cpp
combinedKeys |= keyValue;
```

它是“按位或”运算。

## 6. 为什么要读 5 次再合并

因为电容触摸信号不是绝对稳定的。

可能出现这种情况：

- 第 1 次读到了键 A
- 第 2 次读到了键 A 和键 B
- 第 3 次只读到了键 B

如果只读一次，可能漏掉多键。

现在项目做法是：

1. 连续读 5 次
2. 只要某一位在任意一次读到过，就保留下来

比如：

```text
第1次: 00010000
第2次: 00110000
第3次: 00100000
OR后 : 00110000
```

这样就能把可能同时按下的键更完整地抓出来。

这不是最复杂的滤波算法，但对于这个项目来说，简单、直观，而且够用。

## 7. 从寄存器读到的是什么

`touchPannel.getKeyValue()` 来自 `SC12B.cpp`：

```cpp
uint16_t SC12B::getKeyValue() {
  uint16_t keyvalue = 0xFFFF;
  uint8_t value = 0;
  value = readRegister(REG_OUTPUT1);
  keyvalue = (uint16_t)value << 8 | readRegister(REG_OUTPUT2);
  return keyvalue;
}
```

这段代码的意思是：

1. 先读一个高字节寄存器 `REG_OUTPUT1`
2. 再读一个低字节寄存器 `REG_OUTPUT2`
3. 合并成一个 16 位数 `uint16_t`

为什么要 16 位？

因为二进制的 16 个 bit 很适合表示多个开关状态。每一位都能表示“这个通道是不是被按下”。

## 8. 键值解析是怎么做的

程序不会直接使用寄存器原始位图，而是通过 `parseKeys()` 解析：

```cpp
void parseKeys(uint16_t keyValue, int* pressedKeys, int* keyCount) {
  *keyCount = 0;

  if (keyValue == 0) return;

  for (int i = 4; i < 16; i++) {
    if (keyValue & (1 << i)) {
      int keyIndex = i - 3;

      if (keyIndex >= 1 && keyIndex <= 12) {
        pressedKeys[*keyCount] = keyIndex;
        (*keyCount)++;
      }
    }
  }
}
```

### 8.1 这里在做什么

它会从 bit4 检查到 bit15：

- 如果某一位是 1，说明对应按键被按下
- 再把它换算成 1 到 12 的键号

换算关系是：

```text
bit4  -> key1
bit5  -> key2
bit6  -> key3
...
bit15 -> key12
```

所以 `keyIndex = i - 3`。

### 8.2 为什么不用 bit0 到 bit11

从代码看，这个项目采用的是“SC12B 输出位的第 4 位到第 15 位映射 12 个键”的规则。通常这来自芯片定义或者硬件连线约定。

对初学者来说，你不需要先死记芯片手册，先记住一件事就够了：

“程序已经根据这个芯片的输出格式，把有效位翻译成 1 到 12 的键编号了。”

## 9. 什么是防抖

防抖不只用于机械按键，触摸输入同样可能有抖动。

项目里的防抖逻辑是：

```cpp
if (keyValue != previousKeys && currentTime - lastKeyTime > KEY_DEBOUNCE_TIME)
```

这行判断要求两个条件同时成立：

1. 当前键值和上次不一样
2. 距离上次有效变化已经超过防抖时间

这样做的好处是：

- 避免一次触摸被重复触发很多次
- 避免手指刚放上去时的抖动导致误判

## 10. 检测结果最后去了哪里

当 `parseKeys()` 解析出结果后，会得到：

- `pressedKeys[]`：当前按下的键列表
- `keyCount`：当前按下了几个键

然后主程序会这样处理：

```cpp
displayMultipleKeys(pressedKeys, keyCount);

if (keyCount > 0) {
  playMultipleNotes(pressedKeys, keyCount);
} else {
  stopAllAudio();
}
```

也就是说，按键检测模块本身不负责“发声”和“显示”。它只负责把输入识别准确，然后把结果交给别的模块。

这就是模块化设计的好处：

- 按键模块只管检测
- 音频模块只管发声
- 显示模块只管画界面

## 11. 这个模块里值得学的知识点

### 11.1 I2C 通信

SC12B 和 OLED 都通过 I2C 和 ESP8266 通信。

你可以把 I2C 理解成一条总线，ESP8266 是主机，不同芯片有不同地址。主机想和谁说话，就按地址找谁。

### 11.2 中断思想

中断不是“什么都放进去做”，而是“快速通知主程序”。

这个项目的中断处理方式非常适合初学者学习：

- 中断里只设置标志位
- 主循环里再做真正处理

### 11.3 位运算

`&` 用来判断某一位是不是 1。

`|` 用来把多次结果合并起来。

这是嵌入式里非常常见的写法，因为寄存器、状态字、设备标志都常常是按 bit 保存的。

### 11.4 采样和滤波

连续采 5 次再合并，本质上是一种很简单的抗抖/抗漏检方案。

它说明一个很重要的工程思路：

“真实世界的信号不完美，所以程序通常不会只信一次读数。”

## 12. 如果你想自己改这个功能，可以先改哪里

适合初学者尝试的修改有：

1. 修改 `DEFAULT_TOUCH_SENSITIVITY`，观察触摸灵敏度变化。
2. 修改 `KEY_DEBOUNCE_TIME`，观察按键响应和稳定性的平衡。
3. 修改 `SAMPLE_INTERVAL`，观察系统响应速度。
4. 在串口打印 `keyValue`，看看按不同键时返回的位图数据。

## 13. 一句话总结

这个项目的按键检测，本质上是：

“SC12B 检测触摸 -> ESP8266 通过 I2C 读取寄存器 -> 用位运算解析出按键 -> 通过防抖和多次采样让结果更稳定。”

只要你把这条线理解了，按键检测这部分就算真正入门了。
